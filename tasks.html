Пару слов о требованиям к самим проектам

При выборе проекта есть несколько требований 1 - в команде участвую от 2х до 5 человек (хотя варианты всегда обсуждаемы ). В команде выделяется 1 тимлид - руководитель группы, который кроме написания кода регулирует деятельность группы - раздает задания, ставит сроки, проверяет и т.д. Для каждого проекта будет выделен отдельный репозитарий на GitHub

Теперь о сути задачи. Самое главное при выборе - задача должна нравится. Лучший вариант - если эта задача решает какую-то вашу проблему. Главный вопрос - какую задачу вы хотите реализовать? На фразу "Мы хотим сделать игру" последует логичный вопрос какую? "Мы хотели бы сделать клон "танчиков" с Dendy" - вот пример правильной начальной постановки цели.

P.S. Считайте свои работы - мини-стартапами, денег вряд ли принесут но опыт получите
